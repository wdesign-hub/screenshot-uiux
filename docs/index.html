Pixel-Perfect Screenshot Audit

Capture desktop + iPhone 14 Pro screenshots for design audits

Quick Start View on GitHub →
Built for designers + developers • Playwright powered • Docker ready
Overview

Stop the “it looks different on my machine” debate

Pixel-Perfect Screenshot Audit is a single Python script (screenshot_audit_v2.py) that visits a list of URLs and saves two perfect screenshots for each: one desktop (1440×900) and one mobile iPhone 14 Pro (390×844). It waits for the page to settle, hides cookie banners, runs full-page capture, and names files deterministically so you can drop them straight into Figma for a side-by-side audit.

Why pixel-perfect matters: Design systems break at the edges. Consistent viewports, no browser chrome, same wait conditions, and clean filenames mean your design review compares implementation vs. design — not random scaling, popups, or timing differences.
Features

Everything you need for a clean audit

Desktop + Mobile

Captures both viewports in one run. Fixed sizes: 1440×900 and iPhone 14 Pro 390×844.

Smart Wait

Waits for networkidle by default, with configurable timeouts for slow SPAs.

Hide Banners

Pass CSS selectors to auto-hide cookie notices, chat widgets, and popups.

Figma-ready names

001_homepage, 002_pricing — same names in desktop/ and mobile/ for instant pairing.

Resume support

Re-run any time — skips existing shots and only captures what’s missing.

Docker-ready

No local browser hell. One image with Playwright + Chromium preinstalled.

Quick Start

Get shots in 60 seconds

Build the Docker image

Clone the repo, then build once.

bash
git clone https://github.com/your-org/pixel-perfect-screenshot-audit
cd pixel-perfect-screenshot-audit
docker build -t shots .

Run your first audit

Pass URLs directly. Output goes to ./out.

bash
mkdir -p out
docker run --rm -v $(pwd)/out:/out shots \
  python screenshot_audit_v2.py \
  --urls "https://example.com,https://example.com/pricing" \
  --out /out \
  --hide-selectors "#cookie-banner,.intercom-lightweight-app"

Resume or add URLs

Use --resume to skip existing files. Perfect for large sites.

bash
docker run --rm -v $(pwd)/out:/out shots \
  python screenshot_audit_v2.py \
  --urls-file /out/urls.txt \
  --out /out \
  --resume
Installation

Pick your setup

Option A — Docker (recommended for teams)

Zero dependencies, identical results on Mac, Windows, and CI.

Dockerfile
FROM mcr.microsoft.com/playwright/python:v1.45.0-jammy
WORKDIR /app
COPY requirements.txt .
RUN pip install -r requirements.txt
COPY screenshot_audit_v2.py .
CMD ["python", "screenshot_audit_v2.py", "--help"]

Option B — Local Python

Great for designers who want to run natively. Requires Python 3.10+.

bash
pip install playwright
playwright install chromium
python screenshot_audit_v2.py --urls "https://example.com" --out ./out
Usage

CLI flags

All flags are designed to be explicit. No magic defaults.

FlagDefaultDescription
--urls-Comma-separated URLs. Example: "https://a.com,https://a.com/pricing"
--urls-file-Path to text file, one URL per line. Comments with # supported
--out./outOutput directory (will create desktop/ and mobile/)
--desktop-viewport1440x900Desktop viewport size. Format WIDTHxHEIGHT
--mobile-viewport390x844Mobile viewport (iPhone 14 Pro). Device scale 3 applied
--waitnetworkidleWait strategy: load, domcontentloaded, networkidle
--timeout30000Page load timeout in ms
--hide-selectors-CSS selectors to hide before screenshot (comma-separated)
--auth-state-Path to storage_state.json for authenticated pages
--full-pagetrueCapture full scroll height (set false for viewport only)
--concurrency2Parallel browser contexts
--resumefalseSkip screenshots that already exist
Authentication

Screenshot behind login

For dashboards, admin panels, or paywalled pages, generate a storage_state.json once, then reuse it.

Step-by-step (Playwright Codegen)

Install Playwright

bash
pip install playwright
playwright install chromium

Record your login

This opens a browser. Log in normally, then close the window.

bash
playwright codegen https://app.example.com/login --save-storage=storage_state.json

Use the state in Docker

bash
docker run --rm -v $(pwd):/work -v $(pwd)/out:/out shots \
  python screenshot_audit_v2.py \
  --urls-file /work/private-urls.txt \
  --auth-state /work/storage_state.json \
  --out /out

Non-engineer tip: Keep storage_state.json private — it contains session cookies. Don't commit it to Git.

Output Structure

Figma-ready from the start

Every run creates predictable folders and a manifest for traceability.

out/ ├─ desktop/ │ ├─ 001_example_com.png # 1440×900, full page │ ├─ 002_example_com_pricing.png │ └─ 003_app_example_com_dashboard.png ├─ mobile/ │ ├─ 001_example_com.png # 390×844 @3x, iPhone 14 Pro │ ├─ 002_example_com_pricing.png │ └─ 003_app_example_com_dashboard.png └─ manifest.json # url, timestamp, viewport, filename, hash
Designer workflow: Drag the entire desktop/ folder into Figma, then mobile/. Because names match, they'll stack in order. Create two frames side-by-side and use Figma's "Swap" to compare v1 vs v2 audits.

Troubleshooting

Common issues, fast fixes

Timeouts on slow pages ›

Increase timeout and use domcontentloaded for heavy SPAs:

bash
--wait domcontentloaded --timeout 60000
Blank white screenshots ›

Site likely blocks headless browsers. Add a user agent and slow down:

bash
--user-agent "Mozilla/5.0" --wait networkidle
Login fails with storage_state.json ›

Sessions expire. Re-record with codegen, and ensure you navigate to the app after login before closing. Some sites require 2FA every run — use a test account.

Cookie banners still visible ›

Find the selector in DevTools and pass multiple:

bash
--hide-selectors "#onetrust-banner-sdk,#cookie-banner,.Intercom"
FAQ

Good questions

Can designers run this without Docker? ›

Yes. Install Python 3.10+, run pip install playwright, then playwright install chromium. Double-click a provided run.sh or run.bat that wraps the command.

Does it work with localhost or staging? ›

Absolutely. Use --urls "http://localhost:3000". If running in Docker, use host.docker.internal instead of localhost on Mac/Windows.

How do I get consistent fonts? ›

Docker ensures the same Chromium and fonts everywhere. For local runs, install your brand fonts system-wide, or the tool will fall back to defaults.

Can I screenshot only above-the-fold? ›

Yes, pass --full-page false. You'll get exactly 1440×900 and 390×844 viewport shots.

Is this free for commercial use? ›

Yes, MIT licensed. Built on Playwright. Keep your storage_state.json secure.
